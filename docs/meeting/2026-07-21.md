## 1. 크롤링 안정화 (양현욱)
크롤링을 반복 수행할 때 청크 개수가 달라지는 문제를 분석해 원인을 수정함. 현재는 동일 데이터를 기준으로 청크 수가 안정적으로 유지되도록 개선을 완료함.

이어서 API Key를 외부 환경변수로 분리해 관리하는 작업을 진행 예정이며, 민감정보가 GitHub에 업로드되지 않도록 `.gitignore`에도 반영할 예정임. 보안 강화와 개발·운영 환경 분리가 목적임.

**상태**: 청크 개수 변동 문제 ✅ 해결 완료 / API Key 분리 ▶ 진행예정

---

## 2. 파서 및 FAQ 수집 안정화 (문규남)
FINS FAQ 페이지에서 일부 청크가 정상적으로 생성되지 않는 문제를 분석해 크롤링·파서 수정을 진행 중임. 동시에 운영체제 차이로 FINS FAQ 데이터를 읽지 못하는 문제를 확인해 원인 분석 및 수정을 진행하고 있음.

기존에는 `lxml Parser`만 사용하던 구조였으나, `lxml 실패 시 HTML Parser로 재시도`하는 방식으로 변경함. 이를 통해 운영체제별 호환성이 향상되고 HTML 파싱 실패율이 줄어 FAQ 데이터 수집 안정성이 높아짐.

**상태**: 파서 구조 개선 ✅ 완료 / FAQ 수집·OS 이슈 진단 ▶ 진행중

---

## 3. 검색 성능 개선 — MMR·중복 제한·혼입 방지 (문규남 · 박유상)
검색 단계에 MMR(Maximal Marginal Relevance) 기법을 적용해 중복 검색 결과를 줄이고 다양한 문서가 검색되도록 개선을 진행 중임(문규남). 목적은 Retrieval 품질 개선.

박유상은 검색 결과에서 동일 문서의 여러 청크가 반복 선택되는 문제를 막기 위해 문서당 청크 1개만 선택되도록 제한(Cap)을 적용해 검색 다양성을 높임. 또한 FAQ 검색 시 다른 업무 데이터가 함께 검색되는 문제를 해결하기 위해 Business Function 기반 Hard Filter를 적용해 업무 영역 혼입을 방지하고 검색 정확도를 높임.

**상태**: MMR 적용 ▶ 진행중 / 동일 문서 중복 제한 ✅ 완료 / Hard Filter ✅ 완료

---

## 4. 테스트셋 확장 및 정규화 (박유상)
기존 테스트셋을 약 300개의 자연어 질문으로 확장해 다양한 질문 표현과 실제 사용자 질문을 반영, Retrieval 평가 품질을 높임. 문장 내 중복 공백·줄바꿈·불필요한 공백 등을 정규화해 데이터 일관성도 확보함.

테스트셋 Ground Truth에 대한 라벨 확인과 정답 Chunk 검증도 함께 진행 중임.

**상태**: 자연어 질문 확장 ✅ 완료 / 공백 정규화 ✅ 완료 / 라벨 검증 ▶ 진행중

---

## 5. 메타데이터·코퍼스 보강 (김경은)
문서가 포함된 페이지를 인식해 첨부파일 존재 여부와 개수를 확인함. 첨부문서가 있는 경우 문서명·파일명·문서 양식 등을 저장하는 attachment 필드를 메타데이터에 추가할 예정이며, 파서(parser) 단계에서 관리함. 목적은 첨부문서 검색 지원과 RAG 답변 근거 강화임.

추가로 코퍼스 범위 확장을 위한 URL 추가 수집도 진행할 예정임.

**상태**: 첨부문서 존재 여부 인식 ✅ 완료 / attachment 필드·URL 추가 수집 ▶ 진행예정
