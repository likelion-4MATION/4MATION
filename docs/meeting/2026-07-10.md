## 1. 데이터 분류 & 포함여부 결정 (실전1 핵심)

오늘의 메인 작업. 예솜24 6대 업무 기준 사전 조사 목록을 **실제 사이트 크롤링으로 대조**하며 검색범위를 확정했다.

### 1.1 필수 페이지 URL 매핑 (30건 확정)
- 산출물: `kdic_필수페이지_URL매핑.csv` — 6개 업무별 확정 URL 30개
- 각 행에 `business_function / page_name / url / host / page_type / robots_status / note` 스키마 태깅
- 개요표(33건 주장) 대비 상세표 실측(30건) 기준으로 확정 — 차이 3건은 F4(착오송금) 관련, 문서 불일치로 확인 요청 항목에 등록

### 1.2 분석필요 페이지 포함여부 판정 (12건)
- 산출물: `분석필요태깅_포함여부분석.xlsx` — **①수요 / ②응답가능성(정적·소관) / ③오염리스크** 3축 평가 프레임으로 판정
- 판정 결과 분포:
  - **포함**: 예금자보호제도 FAQ, 착오송금반환지원 신청대상, 상속인 금융거래조회(안내부), 부채증명원(안내부), 채무정보조회 FAQ(D5 대기) 등
  - **링크만**: 예금자보호한도(해외)·보험금 지급대상 검색·신용회복/파산면책/개인회생(타 기관 소관) 등 — 오염리스크 또는 소관 밖
  - **제외**: 부실책임조사 진행현황 조회(로그인·동적·개인정보 동의)
- 판정 근거 예시:
  - 해외 한도 → *해외 수치가 국내 한도 질의에 혼입 위험 + 조회 빈도 극소* → 링크만
  - 타 기관(신복위·법원) 소관 → *기관 업데이트·법 개정 시 오답 전달 위험* → 개요 안내 + 외부 링크만
  - 불법행위신고 → *은닉재산 신고와 유사문구로 혼입 위험 큼* → 챗봇 재확인 후 링크 전달, sub_category 엄격 분리

### 1.3 수집 주의사항·변경사항 정리
- 산출물: `수집_주의사항_및_변경사항.md`
- 핵심 발견:
  1. **표준 XML 사이트맵 없음** → "sitemap 기반 수집" 폐기, 메뉴/브레드크럼 화이트리스트 크롤링으로 대체
  2. **kdic 세션 선행 필요** → 루트 진입 → 세션 확보 → 내부 링크 순회 순서 필수 (무시 시 전 페이지 오류 스냅샷 위험)
  3. **FAQ 두 종류** → kdic `/sp/` 안내형(robots 허용) vs fins `/cm/bbs/` 게시판형(차단·D5) 구분 관리
  4. 동일 항목이 FAQ판/안내판 두 URL로 갈리는 경우(`variant` 컬럼) 페이지별 개별 판단
  5. 크롤러 가드: 오류 페이지(error404, "오류 | KDIC")를 정상 페이지로 오수집하지 않도록 감지 로직 추가

## 2. 이론 근거 정리

### 2.1 파이프라인 & 계층구조 근거 논문
- 산출물: `RAG챗봇_파이프라인_및_계층구조_근거.md`
- **최광미(2024)** — 구현 뼈대(로드→분할→임베딩→벡터DB→검색→프롬프트→API→UI 8단계). 우리 계층구조는 ①②(업무 태그 부여)→④(공간 분리)→⑤(업무 범위 스코핑) 3지점에 개입
- **조준기·양치복(2025)** — 계층 분리가 정확도에 근거한 선택임을 3갈래로 뒷받침:
  1. 메타데이터 필터링이 precision·recall·MRR 상승
  2. 적절한 청킹이 불필요·부정확 정보 혼입 방지
  3. **유사도–정확도 괴리**(상관 -0.882 구간 존재) → 유사도만 높은 타 업무 청크 혼입 문제의 실증 근거

### 2.2 기술 스택 문헌 정리 (검색 파이프라인 근거)
- **BM25** (Robertson & Zaragoza, 2009) — 키워드·수치("착오송금", "1억원") 정밀 매칭 강점, 구어체 약점
- **DPR** (Karpukhin et al., 2020) — 의미 매칭 강점, 수치·고유용어 약점. BEIR(Thakur et al., 2021)로 dense 단독의 zero-shot 취약성 확인
- **RRF** (Cormack et al., 2009) — BM25+dense 융합 근거 (E4 실험)
- **Reranking** (Nogueira & Cho, 2019) — cross-encoder 재정렬, CLOVA Re-ranking Model이 이 계열
- **RAG** (Lewis et al., 2020) + Shuster(2021) — "근거 기반 답변" 요구사항의 직접 배경
