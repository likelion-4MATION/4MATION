## 1. 멘토 확인 대기 항목 — 질문 2가지 재요청

지난 멘토링 답변에 따라 회의 후 FAQ 데이터 수집을 결정했으나, 진행 중 새로 확인이 필요한 사항이 생겨 아래 2가지를 재질문했음.

1. **FAQ 수집 허용 요청 관련**
   fins `/cm/bbs/` 게시판형 FAQ가 `robots.txt` 자체 보안으로 차단되어 수집이 안 되는 상태. 예금보험공사 측에 데이터 수집 허용을 요청드릴 수 있는지, 요청 시 함께 제공해야 할 항목(수집 기한 · 건수 · 범위 등)이 무엇인지 문의함.

2. **stale(구버전) 정보 처리 관련**
   특정 페이지에서 개정 전 수치(5천만원 → 1억원으로 개정되었으나 미반영)가 간헐적으로 노출되는 현상 확인. 이 건도 예금보험공사에 별도 연락이 필요한지, 수집 과정에서 크롤러가 이를 어떻게 처리해야 하는지 문의함.

## 4. 발표 자료 / 대본 준비

발표 구성을 5단계로 확정했음.

1. **배경 문제 설명** — Task 및 해결하려는 문제 소개
2. **전체 파이프라인 설명** — 수집→본문추출→조각내기→임베딩→적재→검색 6단계 거시적 개관 (실전1 범위인 1~5단계 위주)
3. **파이프라인 각 단계별 구조·코드·스크린샷·용어 설명**
4. **산출물 정리 및 평가** (요구사항과 매핑)
5. **실전2와의 연결 및 협업/프로젝트관리 간략 소개**

**세부 결정 사항**
- 2번: 파이프라인을 노드 형태(0-0-0-0-0-0-0)로 표현해 거시적 이미지를 유지한 채, 슬라이드를 넘기며 각 노드에 선으로 부가설명을 이어 보여줌
- 3번: 2번이 제3자도 이해 가능한 친근한 설명이라면, 3번은 같은 주제를 다룬 팀원 수준의 자세한 설명(정확성 위주). 각 슬라이드 상단에 2번의 파이프라인 이미지를 고정 배치해 현재 어느 단계를 보고 있는지 표시
- 4번: 자세한 설명 없이 요구사항 6가지 충족 여부 확인 + 산출물 나열 위주로 간단히 구성
- 5번: 검색 과정(실전2)의 방향성 소개 + 실전1 파이프라인이 정상 동작함을 보여줌. 이어서 협업 방식(노션 문서·자료 공유 · 깃헙 협업 · 디스코드 파일 정리 및 진행상황 공유) 소개

3번(단계별 구조·코드·용어) 대비 자료로 파이프라인 코드 해부 문서 초안을 제작했음 — 각 단계 코드 설명 + 인터랙티브 실습 3종(doc_id 조회기, 청킹 시뮬레이터, RRF 융합 계산기) 포함.
