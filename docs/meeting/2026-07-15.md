## 1. 멘토 답변 반영 — FAQ 수집 및 데이터 최신화 방향

지난 재요청에 대한 멘토 답변을 팀 내 공유하고 다음 두 가지를 확정함.

1. **FAQ 데이터 수집 여부**: 수집해도 된다는 확인 받음.
2. **최신정보 반영 방식**: 부적합 부분이 한 곳뿐이면 수기 수정으로 충분하고, 보호한도처럼 여러 곳에서 반복 등장하는 **범위형 데이터만 별도 테이블로 정합성을 관리**하는 방향이 좋다는 코멘트 받음 — 코퍼스 레벨 플래그 방식 대신 답변 레이어의 별도 참조 테이블로 처리하는 쪽으로 팀 결론 정리함.

## 2. robots.txt 수집 허가 — 승인 획득

fins `/cm/bbs/` FAQ 게시판(H5)에 대해 활용 목적(단순 QA용, 데이터 미노출)을 명시해 예금보험공사에 재요청함. 활용 용도를 특정하지 않았다는 피드백을 반영해 요청을 보완했고, **승인을 받아 수집 완료함**. 챗봇 답변에 실제 반영할지 여부는 실전2 진행 전 별도로 확답하기로 함.

## 3. 사이트 자체 챗봇 분석 & 테스트셋 문제 재확인

KDIC 사이트 자체 챗봇(엔드포인트 2개, 트리 구조)을 직접 조작해 확인함.

- FAQ 질문을 텍스트로 직접 입력했을 때의 답변이 사이트 게시 답변과 다름 확인.
- 버튼 클릭(룰베이스·룩업테이블형)과 텍스트 입력(임베딩 기반으로 추정) 응답도 서로 다름 확인.
- 이 관찰을 바탕으로, 기존 평가셋 52건(FAQ 원문 그대로)만으로는 검증이 자기참조적(self-definition)이 될 위험이 있다는 점을 팀 논의로 재확인함.

## 4. 테스트셋 확장 구성 확정 및 편입

두 개의 소스를 팀이 직접 수집·작성 후 검토해 편입하는 방식으로 확정함 (자동 크롤러로는 수집 불가, 전량 수작업).

- **소스 ①**: 사이트 챗봇에 텍스트로 질문을 넣어 얻은 질문-답변 쌍 수집(예솜24 질문트리 긁어오기).
- **소스 ②**: 기존 FAQ 52건 중 20건을 골라 어휘 표현 자체를 다르게 재구성한 질문 작성(조사 치환 수준이 아닌 실질적 패러프레이즈) — 원문·변형문 쌍 형태로 정리 완료함.


## 5. eval 파이프라인 수정

- 확장분을 `testset_ext_v1.jsonl`로 별도 파일 저장 — 기존 `data/testset.jsonl`(52건)은 그대로 보존.
- `eval.py`가 기존 파일 하나만 읽던 구조에서, 확장 파일도 함께 읽어 합산 채점하도록 코드 수정 완료함.
- 순서: 테스트셋 구축 → 확장 테스트셋 파일 생성 → eval.py 수정 → 재검증.

## 6. 발표 구성 변경 — "검증" 파트 신설

발표자료 논의 결과, 기존 "검색" 설명 파트를 **"검증" 파트로 재편**하기로 결정함.

- 검증 파트는 검색 구조(dense+sparse+hybrid) 설명에 이어, 앞서 확정한 테스트셋 구성(원문형 + 확장 2종)을 포함.
- **발표 순서**: 검증 파트는 발표 구성상 5번째로 배치하기로 함.

## 7. 강사님 논의 — 테스트셋 확장 및 정확도 이슈

**테스트셋 확장 및 1차 검증 결과**
- 확장 테스트셋을 150건 규모로 매핑함.
- 매핑 과정에서 "예금보험금 안내" 문항이 "미수령금" 카테고리에도 걸리는 케이스 발견 — 커버리지 라벨링 보완 필요 확인함.
- 챗봇 응답 정확도 1차 측정: 약 80% 수준. 나머지는 사람이 검수해 판정 보완 필요.
- 시나리오는 6개로 확정 — 평가는 이 6개 시나리오 범위 안에서만 진행.
- 내일까지 확장 테스트셋 완성 목표.

**질문 형태 문제 — 명사형 vs 자연어**
- 수집한 150건이 자연어 문장이 아니라 명사(키워드) 위주로 구성돼 있어 재처리 필요함을 확인함.
- 명사형 질문은 벡터가 짧아 임베딩 거리가 부정확하게 나오는 경우 확인 — 문맥 정보 손실이 원인으로 추정.
- 반면 자연어 문장으로 처리했을 때 정확도가 더 높은 사례도 확인 — 실제 사용자는 명사가 아니라 자연스러운 문장으로 질문하는 경우가 많다는 점을 고려해야 함.
- FAQ에서 가져온 문항은 이미 문장화돼 있어 정확도가 양호함 — 필요 시 FAQ 문장 형식으로 통일하는 방향도 검토(고객사가 원하는 유저 시나리오·응답 형식에 맞출 수 있음).
- 조치: 샘플 일부를 자연어로 변환해 정확도 비교 후, 몇 건은 멘토님께 검증 요청 예정.

**코퍼스 페이지네이션 버그 발견 및 해결**
- FAQ 코퍼스 적재 시 첫 페이지만 읽고 다음 페이지를 못 읽어오던 문제 발견 — 테스트셋 커버리지 부족의 원인 중 하나로 확인.
- 크롤러 수정해 다음 페이지까지 수집 가능하도록 조치 → 정확도 상승 확인(해결).
