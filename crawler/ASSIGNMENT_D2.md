# ASSIGNMENT_D2 — RAG 데모 + 핸드오프 (일)

**목표**: E2E 데모(정답형·링크형·함정형 3종 시연 가능) + 팀이 월요일에 이어받을 수 있는 핸드오프 보고서.
전제: D1 통과 기준 전부 충족.

## 태스크 (순서 고정)

**T1. 라우팅 게이트** (`router.py`) — 검색보다 **먼저** 실행.
- 질문을 `link_registry.json`의 keywords와 매칭 → 매칭 시 임베딩 검색 없이 "소관 아님 안내 문구 + 담당 기관 URL" 응답
- 가볍게: 키워드/간단 규칙 매칭이면 충분, 별도 임베딩 불필요

**T2. RAG 체인 v0** (`rag_chain.py`)
- top-k 검색 → 프롬프트 → LLM 답변 + **출처(page_title · source_url) 표기**
- 프롬프트에 명시: 근거 문서에 없는 내용 답변 금지 · 최신성 주의(2025.9 예금보호 한도 1억원 상향 반영)
- LLM: CLOVA Studio 우선. 인증/연동이 막히면 2시간 룰 — 대체 LLM으로 관통하고 HOLES에 기록

**T3. 저신뢰 가드** — 검색 스코어가 임계 미달이면 단정하지 않고 "확실한 근거를 찾지 못했습니다 + 관련 링크/문의 안내"로 응답. 임계값과 산정 근거를 DECISIONS에 기록.

**T4. 최소 UI** — Streamlit 입력창 1개 + 답변/출처 표시. 막히면 CLI로 대체하고 HOLES 기록.

**T5. 함정 테스트 3종** (`trap_test.py` — 결과를 파일로 남김)
1. **해외 한도 혼입**: "예금 보호 한도 얼마야?" → 국내 답변에 해외 수치 0건
2. **개인회생(타 기관 소관)**: "개인회생 신청 어떻게 해?" → 소관 아님 밝히고 담당 기관 링크 전달 (T1 라우팅 작동 증명)
3. **은닉재산 vs 불법행위 혼동**: 두 개념을 섞은 질문 → 챗봇이 재확인 후 올바른 링크 전달

**T6. (선택, 시간 남으면)** business_function 메타 필터 1차 적용 — 검색 시 업무 태그로 후보를 좁힘. 함정 ①·③ 정확도에 직결.

**T7. 시연 준비** — 정답형 · 링크형 · 함정형 3종 질문 시나리오를 스크립트로 정리, 녹화 또는 라이브 리허설 1회.

**T8. 핸드오프 보고서** (`REPORT_선발대.md`)
- 아키텍처: 파이프라인 다이어그램 + 모듈별 역할
- 의사결정 로그: `DECISIONS.md` 정리본
- 구멍 목록: `HOLES.md` 정리본 + 멘토 확인 대기 4건 (F4 건수 불일치 · fins FAQ 허용 여부 · 분기 항목 FAQ판/안내판 · 해외 한도 링크 크로스체크)
- 팀 분기점: **뚫은 길 넓히기**(P2: 표 보존·Hybrid/RRF·리랭킹·평가셋 / P3: 측정후결정 A/B·robots 건·첨부·fins FAQ) vs **딴 길 탐색**(P4 운영 스택·대안 아키텍처)
- 일요일 밤 통과 조건 4개 예/아니오 판정표

**T9. 마감** — README에 실행 방법(설치→수집→적재→데모) 갱신, 최종 커밋 대상 목록 보고. **git 명령은 실행하지 않는다** — 커밋/푸시는 사용자 직접.
