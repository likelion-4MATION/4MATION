## 보호한도 문항 상세 — "검색 미적중 × 모델 stale" 최악 조합 실측

**재현 확인**: 자연어 의문문 두 문구("예금자 보호 한도는 얼마인가요?" / "예금자보호 한도는 얼마야?") 모두 top-3가 **동일한 3청크**로 점유됨 — `kdic-fins-cm-bbs-selectFaqNramtAply`(미수령금통합신청 FAQ) #37 / #04 / #16. gt 문서(보호한도 · 예금자보호제도 FAQ)는 0/3. 기지 미적중(07-15 ys 세션부터) 그대로.

**점유 청크의 실물 — 신·구 한도 문구가 한 문서 안에 혼재**:
- `#37` (top-1): "ㅇ 보호한도가 **5천만원인 과거에는** … 2천만원 한도로 가지급금을 지급하되…" — 구한도 시절 **과거형 서술**
- `#04` (top-2): "예금자 명의별로(1인당) 원금과 소정의 이자를 합하여 **1억원** 한도 내에서 보호됩니다" — 현행
- `#16` (top-3): 예금담보대출 안내 — 한도 문구 없음

**같은 컨텍스트, 두 번의 HCX 답 (temperature 0.2 고정)**:
- 실측 A — T2 단건 런: *"원리금 합계 1인당 **5천만원**까지 보호됩니다. 단, 예금자가 해당 저축은행에 채무를 부담하고 있는 경우 … 공제한 금액을 대상으로 합니다**[출처3]**."* → 구한도 채택 + 원문의 "과거에는" 한정 **탈락** + 인용 번호는 한도 문구가 없는 #16을 지목(인용 불충실)
- 실측 B — T3 배치 런: *"원리금은 이자에 대한 제세금이 차감되기전 금액 기준으로 1인당 원금과 소정의 이자를 합하여 **1억원** 한도 내에서 보호됩니다**[출처2]**."* → 현행 채택, 인용 정확

**해석**: 검색이 미적중이어도 점유 문서에 신·구 문구가 혼재하면 **RAG 답이 런마다 5천만/1억으로 뒤집히는 롤렛**이 된다 — 저온(0.2)으로도 고정되지 않고, 오답 쪽은 원문의 "과거" 한정까지 벗겨서 현재형으로 재진술한다. 맨몸은 **안정적으로 5천만**(모델 내부 지식 stale, 현행 1억은 2025-09~). 한편 gt 문서 쪽은 ProtSystProtLmts 2청크·ProtSystFaq 17청크가 **일관되게 1억**(page_type 태그도 "보호한도 (1억 상향 핵심 페이지)") — **검색만 맞으면 정답이 안정 공급되는 코퍼스 상태**임을 확인. 즉 이 문항의 병목은 생성이 아니라 검색이다.

## 문항별 근거 인용 (원문 발췌)

- **Q2 (정상 대조의 전형)** — RAG: *"본인명의 공인인증서, 금융인증서나 간편인증 등을 통해 … 신청 바로가기에 접속하여 신청하시거나, … 지급대행지점을 방문하여 신청 … 안내 전화번호(1588-0037)로 문의 [출처1]"* / 맨몸: *"홈페이지 … '예금보험금/파산배당' 메뉴 … 전국 농협은행 및 우리은행 영업점 … 고객상담센터(`****-****`)"* — 맨몸은 메뉴명·기관을 단정 서술하고 전화번호를 자체 마스킹. RAG는 컨텍스트의 실번호를 그대로 통과.
- **Q3 (검색적중 ≠ 근거답변 실증)** — 검색된 3청크의 실제 내용은 *"(온라인) **금융안심포털**에서 미수령금통합신청 … (오프라인) 지급대행점 직접 방문"*. 그러나 RAG 답: *"금융소비자 정보포털 '파인'의 '내 계좌 한눈에' … '어카운트인포' 앱 … 은행이나 우체국 … 사망진단서, 가족관계증명서 … - 출처 : https://www.fss.or.kr/…"* — 파인·어카운트인포·우체국·상속 서류·fss URL 모두 **3청크 어디에도 없음**(오프라인 대조 완료). "참고문서에만 근거하라" 지시가 무시되고 출처 번호 대신 외부 URL을 날조. **적중 문항도 근거 이탈이 가능함을 보여주는 핵심 사례**.
- **Q4** — RAG: *"금융회사를 통한 사전 반환절차를 거치지 않은 경우 공사의 착오송금 반환지원 신청이 불가능합니다 … [출처2]"* (근거 충실). 맨몸: *"5만 원 이상 **1천만 원 이하**의 건"* — 코퍼스 3개 문서가 일관되게 *"건당 5만원 이상 ~ **1억원** 이하"*라고 명시. 보호한도(5천만→1억)에 이어 **맨몸 stale의 두 번째 실측**.
- **Q5** — RAG: *"① 연체 등 신용거래정보가 등록된 자 ② 최저생계비 이상의 수입 … ③ 총채무액 15억 원 이하(무담보 5억, 담보부 10억) [출처2]"* / 맨몸: 제도 취지 설명 + `****-****` — 실행 가능 정보 부재.
- **Q6** — RAG: *"5억원 이하 20% / 5억~20억 1,000만원+15% / 20억~100억 3억2,500만원+10% / 100억 초과 11억2,500만원+5% … 채권금융회사에서 지급"* — 구간표·지급주체 모두 코퍼스 FAQ와 일치(단 "회수기역금액" 오탈자 1건은 HCX 전사 오류, 원문은 "회수기여금액"). 맨몸: *"2023년 현재 … 한도는 20억 원"* — 코퍼스 구간표에 없는 수치(국세청 은닉재산 신고포상금 제도와 혼동 추정).

## 부수 관측 (분류 밖, ⑤ 설계 참고)

1. **맨몸 HCX의 연락처 자체 마스킹**: 전화번호를 `****-****`로 출력 2건(Q2·Q5) — 안내 챗봇으로서 치명적. RAG는 컨텍스트 실번호(1588-0037)를 통과시킴 → 실행 정보는 컨텍스트 주입이 유일한 경로.
2. **맨몸 가짜 URL 2건**: `kdic.or.kr/ko/subsidiary/unclaimed.do`(Q3) 등 실존 경로 패턴과 다른 URL 생성. RAG도 Q3에서 외부 URL 날조 — URL은 모델 생성 금지·출처 필드에서만 조립이 안전.
3. **hybrid `_score` 대역 실측**: 미적중 Q1 top-1 = 0.0310 vs 적중 5문항 top-1 = 0.0315~0.0328. 차이가 0.0005 수준으로 미세 — **절대 임계값으로 적중/미적중 분리 불가**(계약 경고 실측 부합). 저신뢰 reject 설계는 순위 기반/dense 코사인 병기 정의 확정 후에나 가능.
4. **질문 문구 민감성**: T1 스모크의 "예금자보호 한도"(명사구)는 top-3에 gt 2/3 적중, 자연어 의문문 2종은 동일하게 0/3 미적중 — natural .753 < merged .853 격차의 단면.
5. **지연**: RAG가 맨몸 대비 평균 +2.2s(컨텍스트 길이 영향 포함, 최대 7.9s) — 데모 시연 시 체감 요소.

2. **② 프롬프트 근거 준수 강화** — 적중 문항에서도 근거 이탈(Q3)이 나왔으므로 ①과 독립적으로 필요. 저비용·즉시 실험 가능.
