## 미팅 후

**논의사항**
- 데이터 퀄리티를 높일 수 있게 만들기
- 청크 크기를 더 작게 해야 하는지 검토
- 품질보다 일단 끝까지 진행해보기

**논의 결과**
- 시나리오를 복잡하게 구성, 질의에 특정 키워드가 없어도 답변할 수 있도록 개선
- 청크 크기 조정은 보류 → 오늘은 평가 부분에 집중

**검색 지표**
- Hit@10 대신 Hit@3로 평가 → 현재 문서가 38건이라 Hit@10은 정확도 판별에 의미가 없음

**논의 안건**
- 테스트셋 제작 시점: API 키로 생성하면 시간이 오래 걸림 (복잡한 질문을 만들어 돌리면 답변도 복잡하게 나와야 하는데, 문서 찾기에서 끝나면 의미 없음)
- 복잡한 질문 100개 + 어휘가 다른(키워드 미포함) 질문, 규모는 400개 이상
- 문서 반환 방식 결정 필요 → HTML상에서 문서를 뽑아 열람 가능한지, 데이터화·요약은 불필요 → 어떤 질문에 대한 답변으로 나오는 게 중요하지 문서 내용 자체는 없어도 됨 → 문서가 있다고 태깅되면 고객에게 다시 확인 후 반환하는 방식 → API로 LLM 출력 시 답변에 문서를 반환할 수 있는지 확인 필요
- End-to-end로 먼저 진행하고, 실전3에서 정확도를 높이는 프로세스가 맞다고 판단

**리랭커 사용 여부**
- 가벼운 모델을 사용했음에도 300개 처리에 15분 소요, 답변 1건당 약 40초
- 리랭커 추가는 가장 나중으로 미루기 → 답변 시간이 오래 걸리는 반면 정확도 차이가 어느 정도일지 불확실
- 정확도는 크게 상승 (93 vs 이전 87)
- MRR 재랭킹 방법 추가 적용 검토

**질의 분류**
- LLM으로 분류 진행 (정확도가 중요)
