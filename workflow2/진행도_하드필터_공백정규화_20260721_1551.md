## 1. 공백 정규화 (rag.py `classify_query_bf`)

- 문제: 대표문항 "예금자 보호 한도는 얼마인가요?"가 띄어쓰기("보호 한도") 때문에 키워드
  ("보호한도")와 안 맞아 **불명 처리 → 필터 미적용 → 여전히 혼입**.
- 해결: 질의·키워드 양쪽 공백 제거 후 매칭. `q.replace(" ","")` + `k.replace(" ","")`.
- 검증: 질의 분류 커버리지 70%→71%, 정확도 99.1% 유지(신규 오분류 0), 대표문항 → 예금자보호 분류.

## 2. eval_TF.py — hybrid_bf 기준 원인분석판으로 병합

- eval.py의 업데이트(hybrid_bf 모드 + 하드필터 효과 비교)를 이식.
- 핵심 차이: 원인분석·D1판정·미적중·오염체크를 **ANALYZE_MODE=hybrid_bf**(실제 배포 모드) 기준으로 수행.
- 각 미적중에 **원인 태그** 부착: 불명(필터 미적용) / 오분류(엉뚱한 업무) / 정분류(업무 내부 랭킹·청킹).
- 대표문항 표에 dense/hybrid/hybrid_bf 3모드 rank 병기. 산출: `data/error_analysis_TF.md`.
- 발견: 기존 eval.py의 D1판정·미적중이 **hybrid(필터 없는 모드)** 하드코딩이라 필터 효과가 안 보였음.
  hybrid_bf 기준으로 보니 대표 6문항 **6/6 통과**, 플래그십 정상 적중.
