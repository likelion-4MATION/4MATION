# 산출물 정리 — D1 (데이터 파이프라인)

> 선발대 스프린트 D1 기준. 예금보험공사(KDIC) RAG 챗봇 PoC의 데이터 파이프라인 산출물 목록.
> 각 파일이 무엇을 하고 어느 단계를 맡는지 정리한다. 상세 판단 근거는 [`DECISIONS.md`](./crawler/DECISIONS.md), 미해결 이슈·팀 이관은 [`HOLES.md`](./crawler/HOLES.md) 참조.
> 경로는 레포 `crawler/` 하위 기준. 단, ③ 문서 표의 일부는 로컬 작업폴더 전용이다(위치 열 참조).

## 파이프라인 개요

웹페이지 하나가 검색 가능한 형태가 되기까지 6단계를 거친다. 순서 고정 — 앞 단계 산출물이 다음 단계 입력이다.

```
웹페이지(HTML)
  → ① 수집 (crawl)      크롤러가 원본 HTML 확보
  → ② 본문추출 (parse)   규칙 기반으로 본문만 분리
  → ③ 조각내기 (chunk)   검색 단위로 분할 + 메타 부착
  → ④ 임베딩 (embed)     조각을 768d 벡터로 변환
  → ⑤ 적재 (index)      FAISS 벡터스토어에 저장
  → ⑥ 검색 (retrieve)   질의 → 관련 조각 top-k
질문 → 근거 조각 + 출처
```

`pipeline.py`의 `recollect(url)`이 ①~⑤를 URL 단위 원자 함수로 묶는다. `content_hash` 불변 시 전체 스킵.

## ① 코드

| 파일 | 단계 | 역할 |
|---|---|---|
| `config.py` | 설정 | 세션 진입점(`ENTRY_POINTS`)·허용 호스트(`ALLOWED_HOST_SUFFIX`)·차단 오버레이(`POLICY_DISALLOW`)·User-Agent의 단일 원천. 나머지 모듈이 참조 |
| `build_manifest.py` | 준비 | 수집 대상 목록(매니페스트) 생성. 분석필요 xlsx 인자는 선택적(`nargs='?'`) |
| `crawler.py` | ① 수집 | robots 준수 크롤링. 세션 UA로 `robots.txt` 직접 GET→parse (실패 시 fail-open 금지) |
| `parser.py` | ② 본문추출 | 결정론적 BeautifulSoup. GNB 노이즈 제거, 표 값 무손실 직렬화, 브레드크럼 정제, 첨부 링크 목록 보존 |
| `chunk.py` | ③ 조각내기 | 800자 / 오버랩 100. 표 중간분할 금지, FAQ는 Q+A 1쌍=1청크. 11필드 메타 부착 |
| `rag.py` | ④⑤⑥ | 임베딩(ko-sroberta 768d) + FAISS(`IndexFlatIP`) + BM25(kiwipiepy) + RRF(k=60·pool 20). 검색 엔진 |
| `pipeline.py` | 전체 | `recollect(url)=fetch→parse→chunk→upsert(by doc_id)` 원자 함수. 재수집 트리거 프로토타입 |
| `build_testset.py` | 평가 준비 | www FAQ + 질문형 헤딩에서 질의–정답문서 쌍 자동 추출 |
| `eval.py` | 채점 | hit@1 / hit@3 / MRR 산출, dense vs hybrid 비교, MD 리포트 출력 |

## 재생성 (레포 제외 산출물)

무겁고 복원 가능한 산출물은 커밋하지 않는다. 코드·매니페스트·최종 조각만 있으면 재생성된다. **어디서 실행하느냐에 따라 경로가 다르다.**

```bash
# (a) 로컬 작업폴더 — data/raw 캐시가 있는 곳: 무네트워크 재생성
python pipeline.py --use-cache --rebuild

# (b) 레포 클론 — data/raw가 없는 곳: 라이브 재수집으로 재생성 (폴라이트, ≈3~5분 + 최초 모델 다운로드)
python pipeline.py --rebuild
```

- **제외**(재생성 가능): `data/raw/*.html`(32) · `data/meta/*.json` · `data/parsed/*.json`(32) · `data/index/*`
- **포함**(커밋 대상): `data/chunks.jsonl` · `data/testset.jsonl` · `data/eval_report.md` · `data/crawl_report.json`
- 라이브 재수집 시 H6(보호한도 페이지 구버전 간헐 서빙) 확인 필수 — 절차는 [`crawler/README.md`](./crawler/README.md) 빠른 시작 참조.

## 검증 요약 (D1 통과)

| 항목 | 결과 |
|---|---|
